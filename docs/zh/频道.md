# 简介： **通用聊天界面，统一消息中枢**

JiuwenClaw 的​**频道**​，是您与不同聊天平台对话的**渠道**。JiuwenClaw 已经实现了**鸿蒙小艺、飞书**等channel的无缝接入，并持续拓展至更多元渠道。可以直接在通过 **飞书**、**鸿蒙生态终端的小艺app** 应用直接与JiuweClaw对话。

## 数字分身

JiuwenClaw 在**飞书**和**企业微信**频道中支持**群聊数字分身**功能。开启后，机器人将在群聊中作为指定用户的"数字分身"运作——自动识别群聊中与该用户相关的消息并代为回复，以该用户的第一人称口吻进行对话。对于待办、提醒等个人行动类回复，数字分身会自动私发给用户本人，同时在群内补发一条简短确认。无关消息则自动过滤，不消耗 Agent 资源。

该功能默认关闭，开启方式详见下方各频道的配置说明。

配置频道有两种方式：

* **网页配置**​（推荐）— 在 JiuwenClaw前端页面，点击 `智能体` / `频道`卡片，在频道管理模块，填写相关配置。

* **手动编辑 `config.yaml`** — 默认在 `~/.jiuwenclaw/config.yaml` （由首次 `jiuwenclaw-start` 启动时自动生成），将需要的频道设 `enabled: true` 并填好鉴权信息；保存后自动重载，无需重启。

2. 进入**开发者后台**，点击**创建企业自建应用**。

3. 填写应用名称、描述，并上传图标，点击**创建**。

   ![image-20260312202028169](../assets/images/feishu.png)

## 2. 添加机器人能力
1. 在应用配置页面，左侧选择**添加应用能力**。

2. 点击**机器人**下的**添加**按钮。

   ![image-20260312202146446](../assets/images/feishu_add_robot.png)

## 3. 记录机器人应用凭证
1. 进入飞书机器人管理后台

2. 记录 App ID 与 App Secret配置到JiuwenClaw 飞书频道，点击enabled并保存生效

   ![image-20260312202356784](../assets/images/feishu_app_token.png)

   ![image-20260312202651317](../assets/images/feishu_channel_config.png)

## 4. 配置权限
1. 左侧选择**权限管理** -> **API权限**。
2. 搜索并开通以下关键权限（用于收发消息）：
   - `im:message` 相关：以应用的身份发消息、获取用户发给机器人的单聊消息、接收群聊中@机器人消息事件。
   - `im:resource`：上传图片和文件（用于发送图片/文件给用户）。
   - `contact:user.employee_id:readonly`：获取用户ID信息。
   - 也可以批量导入权限，参考火山引擎文档中的权限列表。

![image-20260312203059819](../assets/images/feishu_app_permission.png)

## 5. 配置事件订阅（接收消息）
1. 左侧选择**事件与回调**。
2. **添加事件**：
   -  `im.message.receive_v1`（接收消息事件） 。
   -  `im.message.message_read_v1`（消息已读） 。
3. **添加回调**
   - `card.action.trigger`（卡片回传交互）
4. （可选）**配置加密策略**：如果启用了加密，需要保存 `Encrypt Key`。

![image-20260312203514012](../assets/images/feishu_app_events.png)

3. 提交审核。如果企业开启了免审，版本会立即生效 。
4. 飞书APP中登陆提交应用的账号即可看到发布的聊天机器人

![image-20260312203730114](../assets/images/feishu_app_release.png)

## 9. 开启群聊数字分身（可选）

在完成飞书机器人的基本配置后，可进一步开启数字分身功能，让机器人在群聊中作为指定用户的分身自动回复。

> 飞书中数字分身支持以下触发方式：**@机器人**、**@用户名**（即数字分身对应的用户）、或在消息中**提及用户姓名**，均会触发数字分身回复。

### 前提条件

- 已完成飞书机器人创建、发布，并已将机器人添加到目标群聊（参考步骤 7）

### 配置步骤

1. 在 JiuwenClaw 频道管理页面，进入飞书频道配置，启用 **`group_digital_avatar`** 开关，并配置 **`my_user_id`** 和 **`bot_name`**。

   ![飞书数字分身开关](../assets/images/feishu_group_avatar.png)

2. 配置 **`my_user_id`**（必填）：填入数字分身对应用户的飞书 open_id。获取方式如下：
   - 登录飞书 API 调试台，进入[发送消息接口](https://open.feishu.cn/document/server-docs/im-v1/message/create)
   - 将 `receive_id_type` 选择为 **open_id**
   - 点击**快速复制 open_id**，在弹窗内选择目标用户，获取的 open_id 即为 `my_user_id`

   ![获取飞书open_id步骤1](../assets/images/feishu_user_id_1.png)

   ![获取飞书open_id步骤2](../assets/images/feishu_user_id_2.png)

3. 配置 **`bot_name`**：填写机器人在群聊中的显示名称，用于 @提及识别。

4. （可选）启用 **`enable_memory`** 开关，开启后飞书群聊中可读取及搜索本地记忆文件。

5. 配置**工具/路径权限**：数字分身在群聊中自主运行，无法像私聊那样弹窗让用户确认操作，因此需要预先配置允许使用的工具和可访问的路径。在频道管理页面开启数字分身后，进入权限配置界面，按需设置各工具的权限（`allow` 允许 / `deny` 拒绝）。未配置权限时，需要确认的操作（`ask`）会被自动降级为拒绝（`deny`）。

   ![飞书数字分身权限配置](../assets/images/feishu_group_avatar_permission.png)

也可以通过编辑 `~/.jiuwenclaw/config/config.yaml` 来配置：

``````
channels:
  feishu:
    app_id: "你的 App ID"
    app_secret: "你的 App Secret"
    enabled: true
    # 群聊数字分身配置
    group_digital_avatar: true
    my_user_id: "ou_xxxx"       # 数字分身对应用户的飞书 open_id
    bot_name: "机器人名称"       # 机器人在群聊中的名称
    enable_memory: false         # 是否启用群聊记忆

# 数字分身工具权限配置（按 channel_id + user_id 维度）
permissions:
  owner_scopes:
    feishu:
      "ou_xxxx":                 # 与上方 my_user_id 一致
        defaults:
          "*": "allow"           # 全局默认：allow / deny
        tools:
          bash:
            "*": "deny"          # bash 默认拒绝
            patterns:
              "git status *": "allow"
              "git log *": "allow"
          write:
            "*": "deny"
  deny_guidance_message: "该工具未被授权在数字分身模式下使用。"

6. 飞书机器人需要新增以下权限：
  im:message.group_msg - 获取群组中所有消息（敏感权限）
  contact:contact.base:readonly - 获取通讯录基本信息
  contact:user.base:readonly - 获取用户基本信息
``````

### 配置项说明

| 配置项 | 说明 |
|:------|:-----|
| `group_digital_avatar` | 是否启用群聊数字分身功能。开启后机器人将作为指定用户的数字分身在群聊中活动，智能判断群聊消息是否与该用户相关，自动过滤无关消息、改写相关消息，并对待办/提醒等个人行动类回复自动私发给目标用户，同时在群内补一条简短确认 |
| `my_user_id` | 开启数字分身时**必填**：数字分身对应用户的飞书 open_id（如 `ou_xxx`），未配置则数字分身功能不生效 |
| `bot_name` | 机器人在群聊中的名称，用于 @提及识别 |
| `enable_memory` | 是否启用群聊记忆功能。开启后 JiuwenClaw 在群聊中可读取及搜索本地记忆文件 |
| `owner_scopes` | 数字分身工具权限配置，按 `channel_id` + `user_id` 维度控制各工具的权限。支持 `allow`（允许）、`deny`（拒绝），`ask`（需确认）在数字分身模式下会自动降级为 `deny`。推荐通过网页端配置 |

## 10. 飞书多bot（`feishu_enterprise`）

当一个团队需要在同一套 JiuwenClaw 服务中接入多个飞书应用（多个 Bot）时，可使用 `channels.feishu_enterprise`。

每个 bot 会以独立 channel 运行，channel_id 格式为：`feishu_enterprise:<app_id>`。

当前多bot仅支持手动编辑 `~/.jiuwenclaw/config/config.yaml`：

``````
channels:
  feishu_enterprise:
    bot_a:
      app_id: "cli_xxx"
      app_secret: "xxx"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
    bot_b:
      app_id: "cli_yyy"
      app_secret: "yyy"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
``````

### 字段说明

| 配置项 | 说明 |
|:------|:-----|
| `app_id` | 飞书应用 App ID（必填） |
| `app_secret` | 飞书应用 App Secret（必填） |
| `encrypt_key` | 事件订阅加密 Key（可选） |
| `verification_token` | 事件订阅 Token（可选） |
| `allow_from` | 允许的用户 open_id 列表，空列表表示不限制 |
| `enable_streaming` | 是否启用流式下发 |
| `chat_id` | 固定推送目标（可选） |
| `enabled` | 是否启用该 bot |

### 与单 Bot `feishu` 的区别

- `feishu`：单通道，channel_id 固定为 `feishu`。  
- `feishu_enterprise`：多通道，每个 bot 使用独立 channel_id（`feishu_enterprise:<app_id>`）。  
- 多 Bot 场景下，会按 bot 维度分别记录最近会话信息，避免 bot 之间互相覆盖。

# 钉钉

## 1. 前提条件

- 你必须是**企业管理员**或**拥有开发者权限的成员**的帐号
- 企业已开通**钉钉开发者后台**

## 2. 创建企业内部应用机器人

### 步骤 1：进入钉钉开发者后台

- 访问 [https://open-dev.dingtalk.com](https://open-dev.dingtalk.com)
- 登录后选择 **应用开发 → 企业内部开发 → 创建应用**

![image-20260313090856290](../assets/images/dingding_start.png)

### 步骤 2：填写应用信息

- 应用名称：如 `JiuwenClaw`
- 应用类型：**机器人**

![image-20260313091100590](../assets/images/dingding_create_app.png)

### 步骤 3：添加机器人能力

- 创建成功后，进入应用详情页
- 点击 **应用能力 → 机器人 → 开启机器人配置**
- 填写机器人名称（如 `JiuwenClaw`）、机器人简介（≤10 字）等信息

![image-20260313091410676](../assets/images/dingding_robot_config.png)

### 步骤 4：配置消息接收模式

选择**Stream 模式**，使用WebSocket长连接，无需公网IP，适合云函数/本地调试。


## 3. 配置权限

在 **权限管理** 页面，开启以下权限（根据需求）：
- 送单聊/群聊消息：qyapi_robot_sendmsg
- 根据手机号查 userId（用于单聊）：topapi_v2_user_getbymobile
- 发送互动卡片：Card.Instance.Write
- 流式更新卡片（AI 场景必备）：Card.Streaming.Write

## 4. 发布应用 & 机器人

### 步骤 1：保存并发布机器人

- 在机器人配置页点击 **保存**
- 返回应用主页，点击 **版本管理与发布 → 发布**
- 填写版本号、更新日志、可见范围等信息

![image-20260313091956452](../assets/images/dingding_robot_publish.png)

### 步骤 2：确认发布
- 点击 **确认发布**
- 状态变为 **已发布** 即可使用

> 💡 发布后可在钉钉客户端搜索机器人名称添加到群聊或私聊


## 5. 配置钉钉Channel

从**凭证与基础信息**获取应用凭证 **Client ID**和 **Client Secret**

**JiuwenClaw**服务页面，在“频道-钉钉”中打开Enabled开关，并配置client_id与client_secret并保存

![image-20260313092328435](../assets/images/dingding_channel_enable.png)

1. 在 JiuwenClaw 频道管理页面，进入企业微信频道配置，启用 **`group_digital_avatar`** 开关，并配置 **`my_user_id`** 和 **`bot_name`**。

   ![企业微信数字分身开关](../assets/images/wecom/14_group_avatar.png)

2. 配置 **`my_user_id`**（必填）：填入数字分身对应用户的企业微信账号。获取方式如下：
   - 打开[企业微信管理后台](https://work.weixin.qq.com/wework_admin/login)
   - 进入 **通讯录 -> 组织架构 -> 所在部门 -> 成员详情**
   - 页面中显示的**账号**即为 `my_user_id`

   ![获取企业微信user_id步骤1](../assets/images/wecom/12_user_id_获取.png)

   ![获取企业微信user_id步骤2](../assets/images/wecom/13_user_id_获取_2.png)

3. 配置 **`bot_name`**（可选）：填写机器人在群聊中的显示名称，用于 @提及识别。

4. （可选）启用 **`enable_memory`** 开关，开启后企业微信群聊中可读取及搜索本地记忆文件（群聊中不写入记忆）。

5. 配置**工具/路径权限**：数字分身在群聊中自主运行，无法像私聊那样弹窗让用户确认操作，因此需要预先配置允许使用的工具和可访问的路径。在频道管理页面开启数字分身后，进入权限配置界面，按需设置各工具的权限（`allow` 允许 / `deny` 拒绝）。未配置权限时，需要确认的操作（`ask`）会被自动降级为拒绝（`deny`）。

   ![企业微信数字分身权限配置](../assets/images/wecom/15_group_avatar_permission.png)

也可以通过编辑 `~/.jiuwenclaw/config/config.yaml` 来配置：

``````
channels:
  wecom:
    bot_id: "你的 Bot ID"
    secret: "你的 Secret"
    enabled: true
    # 群聊数字分身配置
    group_digital_avatar: true
    my_user_id: "用户账号"       # 数字分身对应用户的企业微信账号
    bot_name: "机器人名称"       # 机器人在群聊中的名称（可选）
    enable_memory: false         # 是否启用群聊记忆

# 数字分身工具权限配置（按 channel_id + user_id 维度）
permissions:
  owner_scopes:
    wecom:
      "用户账号":                 # 与上方 my_user_id 一致
        defaults:
          "*": "allow"           # 全局默认：allow / deny
        tools:
          bash:
            "*": "deny"
            patterns:
              "git status *": "allow"
              "git log *": "allow"
          write:
            "*": "deny"
  deny_guidance_message: "该工具未被授权在数字分身模式下使用。"
``````

### 配置项说明

| 配置项 | 说明 |
|:------|:-----|
| `group_digital_avatar` | 是否启用群聊数字分身功能。开启后机器人将作为指定用户的数字分身在群聊中活动，智能判断群聊消息是否与该用户相关，自动过滤无关消息、改写相关消息，并对待办/提醒等个人行动类回复自动私发给目标用户，同时在群内补一条简短确认 |
| `my_user_id` | 开启数字分身时**必填**：数字分身对应用户的企业微信账号，未配置则数字分身功能不生效 |
| `bot_name` | 可选：机器人在群聊中的名称，用于 @提及识别 |
| `enable_memory` | 是否启用群聊记忆功能。开启后 JiuwenClaw 在群聊中可读取及搜索本地记忆文件，群聊中不写入记忆 |
| `owner_scopes` | 数字分身工具权限配置，按 `channel_id` + `user_id` 维度控制各工具的权限。支持 `allow`（允许）、`deny`（拒绝），`ask`（需确认）在数字分身模式下会自动降级为 `deny`。推荐通过网页端配置 |

# Telegram

## 1. 创建 Telegram Bot

通过 Telegram 内置的 [@BotFather](https://t.me/BotFather) 创建机器人并获取 Bot Token。

第一步：在 Telegram 中搜索 `@BotFather` 并打开对话

![BotFather 搜索与对话界面](../assets/images/TelegramBotFather.png)

第二步：发送 `/newbot` 命令，按提示输入机器人名称和用户名

BotFather 会依次要求你输入：
- **机器人显示名称**（如 `JiuwenClaw Bot`）
- **机器人用户名**（必须以 `bot` 结尾，如 `jiuwenclaw_bot`）

![BotFather 返回 Bot Token 的消息](../assets/images/Telegram获取BotToken.png)

第三步：**保存 Bot Token**

创建成功后，BotFather 会返回一个格式如 `123456789:ABCDefGhIJKlmN...` 的 Token，请妥善保存，后续配置需要使用。

> ⚠️ **注意**：Bot Token 等同于机器人的密码，请勿泄露。如果 Token 泄露，可在 BotFather 中使用 `/revoke` 重新生成。

## 2. 绑定频道

**方式一：网页配置（推荐）**

在 JiuwenClaw 前端页面，点击 **`智能体` / `频道`** 卡片，在 Telegram 频道模块中填入 Bot Token，开启使用，点击保存即可。

![Telegram 频道配置界面](../assets/images/Telegram频道配置.png)

**方式二：修改 `config.yaml`**

编辑 `~/.jiuwenclaw/config/config.yaml`：

``````
channels:
  telegram:
    # 从 @BotFather 获取的 Bot Token
    bot_token: "你的Bot Token"
    # 用户白名单（Telegram user_id），为空则允许所有用户
    allow_from: []
    # 消息解析模式: Markdown, HTML, None
    parse_mode: Markdown
    # 群聊模式: mention, reply, all, off
    group_chat_mode: mention
    enabled: true
``````

保存后若服务已运行会自动重载，未运行可以执行 `jiuwenclaw-start` 启动服务。

## 3. 配置说明

| 配置项 | 说明 | 默认值 |
|:------|:-----|:------|
| `bot_token` | 从 @BotFather 获取的 Bot Token，**必填** | 空 |
| `allow_from` | 允许使用的 Telegram user_id 白名单列表，为空则允许所有用户 | `[]` |
| `parse_mode` | 消息解析模式：`Markdown`、`HTML` 或 `None`（纯文本） | `Markdown` |
| `group_chat_mode` | 群聊响应模式（见下方详细说明） | `mention` |
| `enabled` | 是否启用 Telegram 频道 | `false` |

### 群聊模式说明

当机器人被添加到 Telegram 群组时，`group_chat_mode` 控制机器人如何响应群内消息：

| 模式 | 说明 |
|:-----|:-----|
| `mention` | **仅响应 @提及**：只有在群内 @机器人 时才会处理消息（推荐） |
| `reply` | **仅响应回复**：只有回复机器人消息时才会处理 |
| `all` | **响应所有消息**：群内所有文本消息都会被处理 |
| `off` | **禁用群聊**：在群组中不响应任何消息 |

## 4. 开始对话

方式1：在 Telegram 中搜索你创建的机器人用户名，发送 `你好` 即可开始对话

![Telegram 中与 Bot 对话界面](../assets/images/Telegram对话界面.png)

方式2：将机器人添加到群组中，根据 `group_chat_mode` 配置与机器人交互

![Telegram 中与 Bot 对话群组界面](../assets/images/Telegram群组对话界面.png)

## 5. 获取 user_id（配置白名单）

如需配置 `allow_from` 白名单，需要先获取用户的 Telegram user_id。可以通过以下方式获取：

1. 在 Telegram 中搜索 `@userinfobot`，发送任意消息即可获取你的 user_id
2. 将获取到的 user_id 填入 `allow_from` 列表中

``````
channels:
  telegram:
    bot_token: "你的Bot Token"
    allow_from:
      - "123456789"    # 用户A的 user_id
      - "987654321"    # 用户B的 user_id
    enabled: true
``````

> 💡 **提示**：`allow_from` 为空列表时，所有用户均可使用机器人。设置白名单后，只有列表中的用户才能与机器人对话。

# Discord

当前版本已经支持 Discord 频道接入。在「频道管理」中配置并启用 Discord Bot，也可以手动编辑 `config.yaml`。

**分步图文（开发者后台创建 Bot、Intent、安装链接、频道管理）见 [Discord.md](Discord.md)。**

- `bot_token`
- `application_id`
- `guild_id`
- `channel_id`
- `block_dm`
- `allow_from`
- `enabled`

可在 `~/.jiuwenclaw/config/config.yaml` 中按如下方式配置：

``````
channels:
  discord:
    bot_token: "Discord Bot Token"
    application_id: "Application ID"
    guild_id: "目标服务器 Guild ID"
    channel_id: "目标频道 Channel ID"
    block_dm: false
    allow_from: []
    enabled: true
``````

## 简略使用说明

1. 在 Discord Developer Portal 创建 Bot，获取 `bot_token`。
2. 在 Bot 页签开启 **Message Content Intent**。
3. 将机器人邀请进目标服务器并赋予读写频道权限。
4. 在 JiuwenClaw 里填入配置并启用 `enabled: true`。

## 字段说明

| 配置项 | 说明 |
|:------|:-----|
| `bot_token` | Discord Bot Token（必填） |
| `application_id` | 应用 ID（可选，建议填写） |
| `guild_id` | 仅监听指定服务器（可选，留空表示不限制） |
| `channel_id` | 仅监听/回复指定频道（可选，留空按消息上下文回复） |
| `block_dm` | 为 `true` 时不处理私信（DM） |
| `allow_from` | 允许访问的 Discord 用户 ID 列表，空列表表示允许所有 |
| `enabled` | 是否启用 Discord 频道 |

## 2. 安卓或IOS接入

### 步骤 1：微信版本升级

- 在微信中依次进入「我」->「设置」->「关于微信」->「版本更新」:

  - 安卓请升级到最新 `8.0.69` 版本。

![wechat_update](../assets/images/wechat_update.png)


### 步骤 2：微信扫码接入

- 打开最新版本的 JiuwenClaw 前端点击「频道」->「微信」，启用微信配置功能并点击保存。页面会出现**二维码**。

![jiuwenclaw_enable_wechat](../assets/images/jiuwenclaw_enable_wechat.png)

- 打开手机微信，通过右上角 「+」->「扫一扫」，扫描上一步骤生成的二维码，继续连接即可完成接入！
